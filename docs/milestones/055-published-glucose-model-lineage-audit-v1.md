# Milestone 055 - Published Glucose Model Lineage Audit v1

This milestone separates three questions that must not be conflated:

1. Can the official PLOS artifact be executed?
2. Can a later author-repository model reproduce the paper's displayed targets?
3. Can the permissively licensed executable currently vendored by Cell do the
   same thing?

The answers are currently **no**, **yes under recovered repository
conditions**, and **no**, respectively. None of these model outputs drive the
simulated hepatocyte state.

## Why The Previous 2/5 Result Was Not Enough

Milestone 054 executed the current author-maintained SBML re-encoding with its
`1.2 mM` external-lactate default and a literal `250 mM` glycogen boundary. It
reproduced the two hormone-phosphorylation endpoints but missed all three
pathway-switch values.

The author's older repository contains a different executable lineage and
additional protocol information:

- the full-model simulation script fixes external lactate at `0.8 mM`;
- it integrates for `200 min`;
- its glycogen grid is nonuniform and does not contain `250 mM`;
- the analysis script chooses the first grid point `>=250 mM`, which is
  `276.6667 mM`;
- the analysis script evaluates the `100 min` output, but the switch roots at
  100 and 200 minutes agree within numerical precision.

No kinetic parameter was fitted, nudged or optimized to obtain the result.

## Four-Way Protocol Matrix

All intervals are derived only from the precision displayed in the paper.
For example, `6.6 mM` is accepted over `[6.55, 6.65] mM`.

| Executable lineage | Lactate | Glycogen | Passed targets |
|---|---:|---:|---:|
| Current permissively licensed re-encoding | 1.2 mM | 250 mM | 2/5 |
| Current re-encoding with recovered repository conditions | 0.8 mM | 276.6667 mM | 2/5 |
| Legacy 2014 author SBML with literal paper label | 0.8 mM | 250 mM | 4/5 |
| Legacy 2014 author SBML with recovered repository conditions | 0.8 mM | 276.6667 mM | 5/5 |

The final legacy-lineage switch predictions are:

| Switch | Paper | Legacy lineage result |
|---|---:|---:|
| HGP / HGU | 6.6 mM | 6.62954 mM |
| Gluconeogenesis / glycolysis | 8.5 mM | 8.45909 mM |
| Glycogenolysis / glycogenesis | 5.1 mM | 5.08009 mM |

The two hormone-phosphorylation targets remain `94.0878%` at `2 mM` glucose
and `5.0997%` at `14 mM` glucose.

## Independent Technical Parity

The author repository also tracks a MATLAB v7.3 result file generated by its
2014 core SBML workflow. Six points spanning the glucose-glycogen grid were
compared for HGP, GNG and GLY against libRoadRunner execution of the legacy
SBML.

- comparison count: `18` flux values across `6` boundary pairs;
- author conversion factor: `12.5 * 60 = 750`;
- maximum absolute difference: `1.377e-9 umol/kg/min`;
- registered tolerance: `1e-8 umol/kg/min`;
- result: pass.

This establishes technical parity between the tracked 2014 MATLAB result and
the 2014 SBML. It does not establish biological validity or exact equivalence
to the 2012 publication run.

## Why Exact Publication Equivalence Is Still Unresolved

The paper's Figure 5 legend states glucose increments of `0.05 mM` and
glycogen increments of `5 mM`. The later author-repository code uses glucose
increments of `0.5 mM` and a nonuniform glycogen grid. The official PLOS SBML
has zero kinetic laws and therefore cannot resolve which executable equations
and exact grid produced the published figure.

The honest conclusion is therefore:

- a later author-model lineage can reproduce all five displayed targets;
- the recovered boundaries explain the legacy lineage's earlier mismatch;
- the current licensed re-encoding is behaviorally different;
- exact execution of the official 2012 artifact remains unavailable.

## License And Distribution Gate

The legacy `glucose-model` repository and its SBML file contain copyright
notices but no explicit reusable license was found. The legacy executable is
therefore **not copied into Cell**. Only checksums, source commits, numerical
audit results and reproduction instructions are retained.

The lineage artifact can be regenerated after separately obtaining the author
repository:

```bash
PYTHONPATH=engine python scripts/audit_koenig2012_lineage.py \
  --legacy-model /path/to/glucose-model/sbml/Koenig2014_Hepatic_Glucose_Model.xml \
  --tracked-result /path/to/glucose-model/results/glucose_glycogen_dependency/glucose_glycogen_dependency.mat
```

Generation requires the optional pinned `libroadrunner==2.9.2` and
`h5py==3.16.0` runtimes. Input SHA-256 checks fail closed.

## Simulation Authority

The browser now reports both scores independently:

- vendored current runtime: `2/5`;
- non-vendored legacy author lineage: `5/5` under recovered conditions;
- exact PLOS artifact: unresolved;
- per-cell rate coupling: blocked.

The lineage audit cannot initialize concentrations, alter reaction rates,
change cell fate, or support a predictive claim. A validated organ-to-cell
scale bridge and independent human validation remain required.
