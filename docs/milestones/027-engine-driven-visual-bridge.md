# Milestone 027 - Engine-driven visual bridge

Status: implemented

M027 connects the TypeScript/Three.js visual scene to the Python engine snapshot
more directly. The browser scene still runs its local visual animation, but when
`/engine-snapshot.json` is available, the readouts, membrane status tint and
organelle glow can be driven by the Python engine's pools, stress axes and
organelle functional-cycle state.

## Language Decision

Keep the visual layer in TypeScript + Three.js for now.

Rationale:

- The current app is already a Vite/TypeScript/Three.js scene.
- Three.js remains the most direct way to keep the cell interactive in the
  browser.
- Python should own biological engine state, model exchange, calibration and
  scientific solvers.
- Rust/C++ remain candidates for later high-performance packet engines,
  PhysiCell integration and heavy solver kernels, not for replacing the current
  browser renderer.
- WebGPU can be evaluated later through Three.js once the scene needs much
  heavier GPU workloads.

## What Was Added

- `src/engineSnapshot.ts`
  - organelle state parsing;
  - stress parsing;
  - flat pool summary;
  - active process summary.
- `src/main.ts`
  - Python organelle ids mapped to visual organelle ids;
  - external engine activity/health can drive organelle glow;
  - Python engine status can tint the membrane;
  - top readouts use Python pools when a snapshot exists;
  - Python engine panel now shows top organelles, stress and active processes.
- `public/engine-snapshot.json`
  - default demo snapshot exported from the Python engine.

## Visual Mapping

- `plasma_membrane` -> membrane
- `cytosol_metabolism` -> glycolysis
- `mitochondria` -> mitochondria
- `nucleus` -> nucleus
- `rough_er` and `smooth_er` -> ER
- `ribosome` -> ribosome
- `golgi` -> Golgi
- `lysosome_endosome` -> lysosome
- `peroxisome` -> peroxisome
- `cytoskeleton` -> cytoskeleton
- `proteasome` remains visible in the Python engine panel until it gets a
  dedicated visual glyph.

## Boundaries

- This is still a snapshot bridge, not live streaming.
- Visual geometry is still generated by the TypeScript scene.
- Python engine values are normalized coarse state, not quantitative clinical
  predictions.
- WebGPU is not adopted in this milestone.
