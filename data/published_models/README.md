# Published Model Artifacts

This directory stores generated validation results for externally published
models. It does not contain fitted project parameters.

## Koenig 2012 Hepatic Glucose Model

Two distinct SBML artifacts are retained under `models/sbml/`:

| Artifact | SHA-256 | Structured audit | Role |
|---|---|---:|---|
| `koenig2012_plos_structure.xml` | `9dc142160a8c4c0179523c438baa6b8f6ba2edc27824a87910fa712ac16c4e6f` | 52 species, 36 reactions, 0 kinetic laws | Exact official PLOS Dataset S2; structure and provenance only |
| `koenig2012_hepatic_glucose_executable.xml` | `5091963c02f39cf00ae02b4fca9362af5f43544851f75b7e17768c7fc56835a3` | 49 species, 258 parameters, 36 reactions, 36 kinetic laws | Author-maintained executable re-encoding; shadow simulation only |

The executable file is pinned from the `sbmlutils` repository at commit
`ad15fdd0eb30e96cba1cdfef9286627eb6d4709c`. The last commit modifying this
specific model file is
`3cf02e91fc6355bdb9971b137454250751e6f808`. The SBML file embeds permissive
redistribution terms; the source repository is MIT licensed. The official PLOS
supplement is distributed with the publication under its open-access terms.

`koenig2012_runtime_validation.json` is generated by:

```bash
PYTHONPATH=engine python scripts/validate_koenig2012_model.py
```

Generation requires the optional pinned runtime `libroadrunner==2.9.2`. The
artifact records the model checksum, runtime version, solver tolerances,
simulation duration, benchmark acceptance basis and raw predictions.

The generated executable re-encoding reproduces the two reported
phosphorylation endpoints but not the three reported pathway-switch glucose
values within intervals implied solely by the publication's displayed
precision. It therefore remains a non-authoritative shadow model and cannot
modify the simulated cell state.

## Model Lineage Audit

`koenig2012_lineage_reproduction.json` records a four-way model/protocol
comparison. A non-vendored legacy 2014 author SBML reaches `5/5` reported
targets only when the later author repository's hidden `0.8 mM` lactate
boundary and actual `276.6667 mM` grid selection are used. The current
permissively licensed executable remains `2/5` under both tested boundary
sets.

The legacy model is not distributed here because no explicit reusable license
was found. Regeneration therefore requires user-supplied paths to the pinned
legacy SBML and tracked MATLAB result. The artifact also records six-point
MATLAB-to-SBML technical parity with a maximum absolute error of `1.377e-9`.

The paper legend and later repository code specify different glucose and
glycogen grids. Exact publication-artifact equivalence therefore remains
unresolved, and all model-lineage results remain disconnected from cell state.
